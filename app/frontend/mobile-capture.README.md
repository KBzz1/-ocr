# mobile-capture

手机端采集页说明。

## 职责

- 扫码进入采集会话。
- 拍照或选择已有图片。
- 展示和调整四边形框选范围。
- 上传原图、框选坐标、页序和图片尺寸。
- 管理完成采集前的页面列表。
- 删除、排序、补拍和重新框选已上传页面。
- 完成采集并提示回到电脑端审核。

## 目标设计稿约束

- 视觉和交互逻辑以 `docs/UI_image/目标设计稿/` 下的手机端未拍照态、框选态、已采集列表态图片为基准。
- 未拍照态使用短状态说明、浅蓝采集卡、单个"拍摄/选择图片"入口、空列表和禁用完成采集。
- 框选态必须支持返回主页面；关键提示、图片、坐标上传说明和主要按钮应在一个手机视口内可操作。
- 四边形只能通过四个圆点手柄拖动；点击图片空白处不得导致角点瞬移。
- 已采集列表使用横向审阅卡片，展示框选后的四边形区域预览、页序、状态、上传时间、删除操作和拖拽排序手柄。

## 不负责

- 自动边界识别算法。
- 裁剪、透视矫正、去噪、去摩尔纹。
- 字段审核和导出。
