# 后端 TDD — 人工审核结果保存

> PRD: PR-BE-008

| ID | 层次 | 行为 | RED 失败点 |
|----|------|------|------------|
| BE-REV-001 | API | 获取审核结果时，若无人工结果则基于字段结果生成初始审核记录 | 首次审核无数据 |
| BE-REV-002 | API | 修改字段保存 `final_value`，保留自动抽取原值和 evidence | 原值或证据被覆盖 |
| BE-REV-003 | 集成 | 人工审核结果与自动字段结果分开存储 | 自动结果被覆盖 |
| BE-REV-004 | API | 确认字段和修改字段更新人工审核状态 | 状态不变 |
| BE-REV-005 | 集成 | 多次修改字段保留修改历史，包含修改前后值和时间 | 历史缺失 |
| BE-REV-006 | API | 重新打开任务优先返回人工 `final_value`，同时保留自动抽取元数据 | 返回自动值或元数据丢失 |
| BE-REV-007 | 单元 | 审核摘要统计未审核、未抽取、可疑、复核失败和无 evidence 字段数量 | 统计错误 |
| BE-REV-008 | API | 单字段可疑、复核失败或质量风险只提示人工核验，不硬阻断保存 | 字段风险被硬阻断 |
| BE-REV-009 | 集成 | 算法整体失败任务不能保存审核、确认或导出，必须返回任务失败错误 | 失败任务仍可被审核 |
| BE-REV-010 | API | 审核完成后任务进入 `done` | 任务完成状态错误 |
