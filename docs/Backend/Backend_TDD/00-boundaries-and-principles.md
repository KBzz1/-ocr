# 后端 TDD — 测试总原则与边界

> 基于 `docs/产品PRD.md` 和 `docs/Shared/` 共享契约编写。

## 当前边界

- 后端负责本地服务、上传文件、任务状态、持久化、审核、导出、日志和隐私保护。
- OCR、图像预处理、裁剪、透视矫正和文档版面解析算法不在本仓库实现。
- 慢阻肺/呼吸系统入院记录专病字段抽取属于当前主代码范围，包括规则分段、提示词编排、字段结果归一化、薄规则质量核验和本地 LLM 调用编排。
- 专病字段抽取不得扩展为通用医学规则引擎，不得生成诊断结论或医学建议。

## 测试原则

- 状态机以 `uploading / processing / review / done / failed` 为准。
- 外部 OCR/文档解析模块缺失或异常时，任务进入 `failed`。
- 慢阻肺字段结果整体不可用、全字段为空或契约非法时，任务进入 `failed`。
- 单字段 evidence 可疑、OCR 风险或复核失败时，字段进入审核页提示人工核验，不直接让整个任务失败。
- 测试优先使用 fixture、手写 OCR 风格样本和可注入 LLM 客户端；不要求真实 OCR、真实图像算法或外部网络。
