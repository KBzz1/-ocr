# 本地日志与隐私保护

> 对应 PRD: PR-BE-010 | 依赖: `docs/Shared/error-codes.md`

```gherkin
Feature: 本地日志与隐私保护
  作为 系统维护者
  我想要 记录系统关键事件日志但不泄露患者隐私
  以便 在出现问题时快速定位，同时满足数据安全要求

  Scenario: 记录系统关键事件
    Given 系统正在运行
    When 发生以下任一事件：
      | 事件               |
      | 系统启动            |
      | 采集会话创建         |
      | 图片上传            |
      | 完成采集            |
      | 任务处理开始/完成/失败 |
      | 字段审核保存         |
      | 导出成功/失败        |
    Then 日志中应记录对应事件，包含时间戳和事件描述

  Scenario: 错误日志包含必要上下文
    Given 任务 T001 处理失败
    When 我查看错误日志
    Then 日志应包含 task_id 或 session_id
    And 日志应包含 error_code 和简短错误原因

  Scenario: 日志不记录敏感信息
    Given 任务 T001 的病历中包含患者姓名 "张三" 和身份证号 "110101199001011234"
    When 系统记录处理日志
    Then 日志中不应出现完整的病历原文
    And 日志中不应出现身份证号、手机号等个人信息
    And 日志中不应出现图片 base64 编码

  Scenario: 日志仅保存在本地
    Given 系统已运行一段时间产生日志
    When 检查系统网络请求
    Then 日志不应被上传到任何外部服务
    And 日志文件仅存在于本地日志目录

  Scenario: 日志轮转防止无限增长
    Given 日志文件已超过配置的大小上限
    When 系统继续写入日志
    Then 旧日志应被轮转或清理
    And 新日志写入不应因磁盘空间问题而崩溃
```
