# MVP 术语表

| 术语 | 说明 |
|------|------|
| 工作站 | 运行在本地 Windows 电脑上的后端服务 + 前端页面 |
| 任务 | 一次病历图片上传、OCR/结构化处理、人工审核和导出的完整记录 |
| 手机上传入口 | 电脑端创建任务后生成的二维码链接，手机扫码后进入该任务的上传页 |
| 上传令牌 | 手机上传接口使用的轻量校验值，用于确认图片上传到指定任务 |
| 采集二维码 | 电脑端为任务生成的二维码，手机扫码后进入多图上传页 |
| 采集人 | 手机端操作用户，负责拍照或选择图片并上传（通常与医生为同一人） |
| 医生 | 电脑端操作用户，负责审核、确认和导出 |
| 页面图像 | 手机端上传的一张病历图片；多张图片按上传成功顺序作为页序 |
| OCR 文本 | 本地 OCR/文档解析模块从图片中识别出的文本 |
| 结构化字段 | 从病历文书抽取出的字段名-字段值对，如"主诉：头痛3天" |
| Schema | 结构化字段定义文件，含版本号和文书类型标识 |
| 算法模块 | 本地 OCR、文档解析和结构化字段抽取模块；本仓库只调用，不实现算法 |

MVP 不再使用"采集会话"和"四边形框选"作为核心业务概念。
