# 病历图片 OCR 结构化 MVP 工具 — 产品 PRD

## 1. 产品定位

本产品定位为**医生个人使用的本地 OCR 结构化小工具**。

- **运行环境**：医生本地 Windows 电脑，手机仅作为拍照上传终端。
- **核心流程**：电脑端新建任务并展示二维码 → 手机扫码拍照/选择多张图片上传 → 后端调用本地 OCR 和结构化抽取模块 → 电脑端人工审核、修正、保存和导出。
- **算法依赖说明**：OCR、文档解析、结构化字段抽取由本地算法模块提供。本仓库只负责调用、保存结果、展示审核和导出。
- **边界约束**：不接入 HIS/EMR，不写回病历系统，不生成诊断结论或医学建议。
- **网络边界**：手机与电脑只在本地局域网或电脑热点内通信，不依赖云 API、CDN、遥测或运行时联网下载。

**核心价值**：把"逐字录入"变成"拍照上传 + 电脑端核对修正"。

---

## 2. MVP 收敛范围

MVP 保留：

- 手机扫码拍照上传。
- 多张图片归入同一个任务。
- 图片页序按上传成功顺序确定。
- 本地 OCR/结构化处理。
- 电脑端审核结构化字段。
- JSON/Excel 导出。
- 三个电脑端页面：工作台总览、任务管理、审核界面。

MVP 不做：

- 独立采集会话。
- 会话过期、锁定、取消、解锁或修订采集。
- 四边形框选、重新框选、裁剪、透视矫正、自动边界识别。
- 手机端拖拽排序、补拍替换、复杂页面管理。
- 手机端审核、字段展示或导出。
- 多医生协同、云同步、医院系统接口。
- 复杂字段质控状态和导出前重预警流程。

---

## 3. 核心目标

1. **手机快速上传**：医生用手机扫码后拍照或选择多张病历图片，直接上传到同一个任务。
2. **本地处理**：本地电脑调用 OCR 和结构化抽取模块，生成 OCR 文本和结构化字段初稿。
3. **人工审核**：电脑端展示原图、OCR 文本和结构化字段，允许医生快速修改和保存。
4. **结构化导出**：支持导出审核后的 JSON 和 Excel。
5. **轻量开发**：优先完成可用 MVP，复杂采集和质控能力后置。

---

## 4. 用户角色

| 角色 | 职责 |
|------|------|
| 医生/病案整理人员 | 新建任务、手机拍照上传、审核字段、导出结果 |
| 系统维护者 | 放置本地模型和配置，启动/停止本地工具，排查本地运行问题 |

手机端用户与电脑端用户通常是同一人。手机端只负责图片上传；电脑端负责处理、审核和导出。

## 5. 业务主流程

```text
启动本地工具
    ↓
电脑端点击"新建任务"
    ↓
系统创建任务，状态为 uploading，并生成手机上传二维码
    ↓
手机扫码进入上传页
    ↓
手机拍照或选择图片
    ↓
上传一张或多张图片，页序按上传成功顺序确定
    ↓
手机端点击"完成上传"
    ↓
任务进入 processing，后端调用本地 OCR/结构化模块
    ↓
处理成功后任务进入 review
    ↓
电脑端人工审核、修正、保存字段
    ↓
任务标记 done，并导出 JSON/Excel
```

失败流程：

- 图片上传失败：手机端提示失败，用户可重新选择或重新拍摄后上传。
- 完成上传时没有图片：返回 `TASK_EMPTY`，任务留在 `uploading`。
- OCR/结构化模块未配置、异常、返回空字段或契约非法：任务进入 `failed`，任务管理页展示原因。
- 图片质量或页序错误：MVP 建议新建任务重新上传；后续版本再考虑补传、替换、排序。

## 6. 任务状态

任务状态统一使用 `docs/Shared/state-enums.md`：

```text
uploading -> processing -> review -> done
                       \-> failed
```

状态含义：

| 状态 | 含义 |
|------|------|
| `uploading` | 任务已创建，手机端正在上传图片 |
| `processing` | 后端正在调用本地 OCR/结构化模块 |
| `review` | 处理完成，等待电脑端人工审核 |
| `done` | 审核完成，可查看和导出结果 |
| `failed` | 上传完成、处理或导出前置步骤失败 |

允许操作：

- `uploading`：继续上传图片、完成上传。
- `processing`：查看处理进度，不允许修改输入图片。
- `review`：审核字段、保存修改、重新处理、标记完成、导出。
- `done`：查看结果、导出、必要时重新处理。
- `failed`：查看失败原因、重新处理；如果图片输入错误，建议新建任务。

## 7. 前端产品需求

### PR-FE-001：工作台总览

工作台总览是电脑端入口，聚焦三个动作：新建任务、查看最近任务、进入任务管理。

首页应包含：

- 系统状态：显示本地服务是否可用、OCR/结构化模块是否配置。
- 新建任务入口：点击后创建 `uploading` 任务并弹出二维码。
- 手机上传二维码弹窗：展示二维码、任务编号、已上传图片数量和"手机无法连接？"帮助提示。
- 最近任务：展示最近任务编号、创建时间、图片数量、状态和主要操作。
- 任务统计：展示 `uploading`、`processing`、`review`、`done`、`failed` 数量。
- 主导航：提供"工作台总览"和"任务管理"入口，审核界面从具体任务进入。

验收标准：

- 双击启动脚本后浏览器可打开工作台总览。
- 点击"新建任务"后创建 `uploading` 任务并展示二维码。
- 二维码只用于手机上传该任务图片，不代表独立会话。
- 首页能展示最近任务和状态统计。
- 不展示会话过期、锁定、修订采集、四边形框选相关入口。

### PR-FE-002：手机上传页

手机上传页只负责多图上传和完成上传。

核心操作：

1. 扫码进入任务上传页。
2. 点击"拍照/选择图片"调用手机浏览器拍照或相册选择能力。
3. 上传图片到当前任务。
4. 展示已上传图片列表，按上传成功顺序显示页码。
5. 可继续添加图片。
6. 点击"完成上传"，任务进入处理流程。

手机端不负责：

- 四边形框选。
- 拖拽排序。
- 补拍替换某页。
- 审核字段。
- 展示 OCR 文本或结构化结果。

验收标准：

- 手机浏览器可扫码进入上传页。
- 可拍照或选择本地图片上传。
- 多张图片按上传成功顺序归入同一个任务。
- 上传列表展示页码、上传状态和缩略图或文件名。
- 完成上传前至少需要 1 张图片。
- 点击"完成上传"后提示回到电脑端查看处理结果。

### PR-FE-003：任务管理

任务管理页展示所有任务，并提供筛选和操作入口。

列表字段：

- 任务编号。
- 创建时间。
- 图片数量。
- 当前状态。
- 错误原因（仅失败任务）。
- 操作按钮。

筛选：

- 全部。
- 上传中。
- 处理中。
- 待审核。
- 已完成。
- 失败。

操作：

- `uploading`：查看二维码。
- `processing`：查看进度。
- `review`：进入审核、重新处理、导出。
- `done`：查看结果、导出、重新处理。
- `failed`：查看原因、重新处理。

验收标准：

- 新建任务后任务列表出现 `uploading` 任务。
- 完成上传后状态进入 `processing`。
- 处理成功后状态进入 `review`。
- 审核完成后状态进入 `done`。
- 失败任务展示可理解的错误原因。

### PR-FE-004：审核界面

审核界面是 MVP 的核心工作界面。

布局建议：

- 左侧：任务图片列表和当前图片预览。
- 中间：OCR 文本，可按页查看或查看合并文本。
- 右侧：结构化字段表单。

字段能力：

- 字段值可编辑。
- 保存人工修改。
- 可标记字段已确认。
- 可一键保存当前审核结果。
- 可标记任务完成。
- 可导出 JSON/Excel。

MVP 字段状态：

- `unreviewed`：未审核。
- `confirmed`：已确认。
- `modified`：已修改。

验收标准：

- 审核页能查看原图、OCR 文本和结构化字段。
- 字段修改后可保存，刷新后不丢失。
- 审核完成后任务可进入 `done`。
- 导出内容来自人工审核后的最终值。

### PR-FE-005：导出与错误提示

导出能力：

- 支持 JSON 导出。
- 支持 Excel 导出。
- 导出内容使用人工审核后的最终字段值。

错误提示：

- 手机无法连接电脑。
- 图片上传失败。
- 图片格式不支持或过大。
- OCR/结构化处理失败。
- 导出失败。

验收标准：

- `review` 和 `done` 任务可导出。
- 导出失败不影响已保存审核数据。
- 错误文案面向医生，不暴露调用堆栈。

## 8. 后端产品需求

### PR-BE-001：本地服务启动

后端以本地服务形式运行在 Windows 电脑上，提供前端页面、API、文件上传、任务处理和导出能力。必须支持离线运行。

验收标准：

- 断网下系统可启动。
- 电脑端可访问工作台。
- 手机在同一局域网或电脑热点下可访问上传页。

### PR-BE-002：任务创建与手机上传入口

电脑端点击"新建任务"时，后端创建 `uploading` 任务，并生成手机上传 URL 或二维码内容。手机上传直接绑定任务，不再创建独立采集会话。

任务记录至少包含：

- `task_id`
- `status`
- `created_at`
- `upload_token`
- `images`
- `error`

验收标准：

- 新建任务返回任务 ID 和手机上传 URL。
- 手机上传接口校验任务 ID 和上传令牌。
- 非 `uploading` 任务拒绝继续上传图片。

### PR-BE-003：图片上传与文件管理

后端接收手机上传的图片，保存到任务独立目录。多图页序按上传成功顺序确定。

验收标准：

- 支持 jpg/jpeg/png/bmp 等常见图片格式。
- 拒绝非图片、伪装文件和超过大小限制的图片。
- 上传图片不会覆盖其他任务文件。
- 每张图片保存原图路径、页序、上传时间和可访问预览路径。

### PR-BE-004：任务生命周期管理

后端维护 `uploading / processing / review / done / failed` 五个任务状态。

处理规则：

- `uploading` 任务完成上传后，如果已有图片，则进入 `processing`。
- `processing` 调用本地 OCR/结构化模块。
- 处理成功进入 `review`。
- 审核完成进入 `done`。
- 处理失败进入 `failed`，保存错误码和错误说明。

验收标准：

- 状态转换符合 `docs/Shared/state-enums.md`。
- 非法状态转换返回 `INVALID_TASK_TRANSITION`。
- 失败任务保留错误信息。
- 算法模块缺失、异常、返回整体无效/全空/无法解析输出或契约非法时进入 `failed`。单字段抽取为空或不确定不触发整任务失败，而是进入 `review` 由人工核验。

### PR-BE-005：OCR 与结构化抽取集成

OCR 和结构化抽取能力由本地算法模块提供。后端负责传入任务图片、接收 OCR 文本和结构化字段候选，并持久化结果。

输入：

- 任务图片列表，按上传顺序排列。
- 当前字段 Schema。

输出：

- 每页 OCR 文本。
- 合并 OCR 文本。
- 结构化字段候选（当前 MVP 字段来源为慢阻肺专病 schema）。
- 字段抽取元数据：`extraction_status`、`verification_status`、`quality_flags`、`ocr_correction`。
- 可选来源证据和置信度。

验收标准：

- 可调用本地模块生成 OCR 文本和结构化字段。
- 模块不可用、调用失败或返回整体无效/全空/无法解析输出时任务进入 `failed`。
- 单字段抽取为空或不确定时任务进入 `review`，前端展示字段风险供人工核验。
- 后端不得基于规则、模板或 schema 自行补造字段。

### PR-BE-006：Schema 管理

当前 MVP 使用慢阻肺专病 schema 定义结构化字段范围。后续可扩展为多个文书类型。

验收标准：

- 任务记录使用的 schema 版本。
- 前端可根据 schema 展示字段组。
- schema 只定义字段范围，不生成字段值。

### PR-BE-007：审核结果保存

审核结果与自动抽取结果分开保存，避免覆盖原始候选结果。

支持：

- 读取自动候选和人工结果。
- 保存单字段或整组字段的人工最终值。
- 标记字段状态为 `unreviewed`、`confirmed` 或 `modified`。
- 标记任务完成。

验收标准：

- 修改字段后可保存人工结果。
- 再次打开任务显示人工修改后的内容。
- 仍可追溯自动抽取原值。

### PR-BE-008：导出服务

基于人工审核结果生成导出文件。MVP 支持 JSON 和 Excel。

验收标准：

- `review` 和 `done` 任务可导出。
- 导出内容来自人工最终值。
- 导出文件保存到本地 `exports/`。
- 导出失败返回 `EXPORT_FAILED`，不破坏审核数据。

### PR-BE-009：本地日志与错误追踪

记录启动、上传、处理、审核、导出和失败原因。日志只保存在本地，不上传。

验收标准：

- 出错时能查到任务 ID、错误码和简短原因。
- 日志不包含完整病历原文、身份证号、图片 base64 或模型输出全文。

## 9. 核心数据实体

| 实体 | 核心属性 |
|------|----------|
| **Task（任务）** | 任务标识、状态（uploading/processing/review/done/failed）、创建/更新时间、图片数量、上传令牌、错误信息 |
| **PageImage（页面图像）** | 所属任务、页序号、原始图像路径、预览路径、尺寸、上传时间 |
| **DocumentResult（OCR 结果）** | 所属任务、逐页 OCR 文本、合并全文、创建时间 |
| **StructuredField（结构化字段）** | 字段 key、字段名、自动抽取值、人工最终值、审核状态、可选来源证据和置信度 |
| **ReviewResult（审核结果）** | 所属任务、逐字段最终值、审核状态、修改时间 |
| **ExportRecord（导出记录）** | 所属任务、导出格式、文件路径、导出时间 |

## 10. API 设计概要

| 模块 | 主要能力 |
|------|----------|
| **系统** | 查询本地服务状态和算法模块配置状态 |
| **任务管理** | 创建任务、任务列表、任务详情、删除任务、重新处理 |
| **手机上传** | 上传图片、完成上传、查询任务上传状态 |
| **处理结果** | 获取页面图像、OCR 文本、结构化字段 |
| **人工审核** | 获取审核结果、保存字段、确认字段、标记任务完成 |
| **导出** | 导出 JSON、导出 Excel |

建议 API：

```text
POST /api/tasks
GET  /api/tasks
GET  /api/tasks/{task_id}
POST /api/mobile-upload/{task_id}/images
POST /api/mobile-upload/{task_id}/finish
POST /api/tasks/{task_id}/process
GET  /api/tasks/{task_id}/review
PUT  /api/tasks/{task_id}/review
POST /api/tasks/{task_id}/complete
GET  /api/tasks/{task_id}/export/json
GET  /api/tasks/{task_id}/export/excel
```

## 11. 前后端职责边界

| 层 | 职责 | 不负责 |
|----|------|--------|
| **前端** | 任务入口、手机图片上传、任务列表、原图/OCR/字段展示、人工编辑、导出触发 | OCR、字段抽取、图像处理、字段补造 |
| **后端** | 文件接收、任务状态、调用本地算法模块、结果持久化、审核保存、导出生成 | 复杂手机采集交互、OCR/LLM/图像算法实现 |
| **算法模块** | OCR、文档解析、结构化字段抽取 | 任务管理、人工审核、导出 UI |

前后端通过 API 解耦。算法模块作为本地依赖集成，替换或升级算法模块不应影响前端核心流程。
