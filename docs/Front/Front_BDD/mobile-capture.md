# 手机端文档采集

> 对应 PRD: PR-FE-002 | 依赖: `docs/Shared/terminology.md`, `docs/Shared/state-enums.md`

```gherkin
Feature: 手机端文档采集
  作为 采集人
  我想要 用手机拍摄病历文书页面并上传到工作站
  以便 在电脑端进行结构化审核

  Scenario: 通过有效会话链接进入采集页
    Given 存在一个有效的采集会话 "sess_001"
    When 我通过 URL "http://192.168.1.100:{port}/capture?session=sess_001" 访问采集页
    Then 我应该看到采集页面
    And 我应该看到 "已采集 0 页" 的提示
    And 我应该看到 "拍摄/选择图片" 按钮
    And 页面应展示绿色 "采集会话进行中" 状态

  Scenario: 未拍照态按目标设计稿展示轻量采集入口
    Given 我通过有效会话链接进入采集页
    And 当前会话尚未上传任何页面
    When 页面加载完成
    Then 顶栏应显示返回按钮、标题 "病历文书采集" 和帮助按钮
    And 顶部应显示绿色 "采集会话进行中" 胶囊
    And 胶囊右侧只显示一行短说明 "请拍摄病历文书页面，完成后回到电脑端审核"
    And 主卡片应显示 "请拍摄病历文书页面"、已采集 0 页和拍摄引导插画或本地静态图
    And 主卡片应提供单个 "拍摄/选择图片" 主按钮
    And 不应展示多段采集教程、开发者参数、IP、端口或完整 URL

  Scenario: 零页列表按目标设计稿展示空状态和禁用完成采集
    Given 我在有效会话的未拍照态
    When 我查看 "已采集页面列表"
    Then 列表标题右侧应显示 "管理顺序" 入口或列表图标
    And 列表内容应显示 "暂未上传页面" 空状态
    And 空状态说明应为 "上传后可在这里查看、删除或调整顺序"
    And "完成采集" 按钮应该禁用
    And 我应该看到 "至少上传 1 页后可完成采集" 的提示

  Scenario: 会话不存在时禁用采集功能
    Given 会话 "sess_invalid" 不存在
    When 我通过 URL 访问 "?session=sess_invalid"
    Then 我应该看到 "无效的采集链接，请重新扫描二维码"
    And "拍摄/选择图片" 按钮应该处于禁用状态

  Scenario: 会话过期后禁止新增内容
    Given 采集会话 "sess_001" 已过期
    When 我访问该会话的采集页
    Then 我应该看到 "采集会话已过期" 的提示
    And 所有拍摄/选择、上传、删除、排序、补拍、重新框选按钮都应该处于禁用状态

  Scenario: 已锁定的会话禁止编辑
    Given 采集会话 "sess_001" 状态为 locked
    When 我访问该会话的采集页
    Then 我应该看到 "采集已完成，请在电脑端查看" 的提示
    And 拍摄/选择图片、删除、排序、补拍、重新框选入口均不可用

  Scenario: 拍摄或选择图片后进入框选页
    Given 我在采集页且会话有效
    When 我点击 "拍摄/选择图片" 按钮
    Then 应该触发设备的相机或文件选择
    When 我选取一张图片
    Then 我应该进入 "调整识别范围" 页面
    And 我应该看到该图片的本地预览
    And 我应该看到四边形框选 overlay 叠加在图片上
    And 我不应该看到 X/Y 像素值输入框或坐标滑块

  Scenario: 框选页按目标设计稿在多宽度下保持单屏关键操作
    Given 我已选择一张图片并进入 "调整识别范围" 页面
    When 我分别使用 360px、375px、390px、414px、428px 宽度的手机视口查看页面
    Then 顶栏应显示返回按钮、标题 "调整识别范围" 和帮助按钮
    And 顶栏下方应显示一条浅蓝提示 "请框选病历正文区域，排除屏幕边缘、灰色背景和工具栏"
    And 图片预览应在提示条和按钮区之间完整展示主要内容
    And 图片下方应显示 "将上传原图与框选坐标"
    And 底部操作区应同时可见 "重拍"、"重新框选"、"确认上传"
    And 我不需要向下滚动就能看到上述关键操作

  Scenario: 框选页返回按钮回到进入前的采集主页面
    Given 我从未拍照态选择图片进入 "调整识别范围" 页面
    When 我点击顶栏返回按钮
    Then 我应该回到 "病历文书采集" 主页面
    And 页面仍应显示已采集 0 页和拍摄入口
    When 我已上传 1 页并从列表点击 "重新框选" 进入 "调整识别范围" 页面
    And 我点击顶栏返回按钮
    Then 我应该回到 "已采集页面" 列表
    And 原已上传页面不应被删除、替换或重新排序

  Scenario: 从相册选择图片作为病历页面
    Given 我在采集页且会话有效
    When 我点击 "拍摄/选择图片" 并从系统入口选择相册图片
    And 我选择一张 jpg 图片
    Then 我应该进入 "调整识别范围" 页面
    And 我应该看到四边形框选 overlay

  Scenario: 选择 PDF 文件时前端直接拦截
    Given 我在采集页且会话有效
    When 我点击 "拍摄/选择图片" 并选择一个 pdf 文件
    Then 我应该看到 "不支持的文件类型" 的错误提示
    And 不应该调用上传 API

  Scenario: 选择超过 20MB 的图片时前端拦截
    Given 我在采集页且会话有效
    When 我选择一张超过 20MB 的图片
    Then 我应该看到 "图片过大（最大 20MB）" 的提示
    And 不应该调用上传 API

  Scenario: 上传过程中显示加载状态并阻止重复点击
    Given 我在采集页已选择一张图片并确认框选
    When 我点击 "确认上传"
    Then 该页面应该显示上传中的 loading 状态
    And 我再次点击 "确认上传" 时不应该产生第二个上传请求

  Scenario: 上传成功后回到页面列表并显示行内操作
    Given 我在框选页已确认上传一张图片
    When 上传成功
    Then 我应该回到已采集页面列表
    And 该页应该显示缩略图、"已上传" 状态、页序
    And 该页右侧应该显示 "补拍"、"重新框选"、"删除" 行内按钮

  Scenario: 上传成功后列表缩略图展示框选后的四边形区域
    Given 我在框选页将四个角点调整为病历正文区域
    When 我确认上传且上传成功
    Then 我应该回到已采集页面列表
    And 对应页面缩略图应展示本次框选后的四边形区域预览
    And 缩略图不应直接展示未框选的整张原图作为唯一预览
    And 前端不应展示透视矫正、OCR 识别或字段抽取结果

  Scenario: 单页上传失败后只重试失败页
    Given 我已成功上传第 1 页
    And 第 2 页上传时网络异常导致失败
    When 我查看页面列表
    Then 第 2 页应该显示 "上传失败，请重试"
    And 第 2 页只应该显示 "重试" 和 "删除" 按钮
    And 第 1 页不应该被重复上传
    When 我点击第 2 页的重试按钮
    Then 应该只重新上传第 2 页

  Scenario: 零页时不允许完成采集
    Given 我在采集页且未上传任何页面
    When 我点击 "完成采集" 按钮
    Then 我应该看到 "请至少采集一页病历" 的提示
    And 不应该调用 finish API

  Scenario: 完成采集后锁定会话并提示到电脑端查看
    Given 我已成功上传 3 页病历图片
    When 我点击 "完成采集" 按钮
    Then 系统应该调用 POST /api/mobile/sess_001/finish
    And 页面应该切换为只读完成态
    And 我应该看到 "采集完成，请在电脑端继续审核" 的提示

  Scenario: 移动端视口下按钮和交互区域不遮挡
    Given 我分别使用 360px、375px、390px、414px、428px 宽度的移动设备视口
    When 我访问采集页
    Then 所有主要操作按钮应该在可视区域内
    And 按钮不应该相互遮挡
    And 四边形角点手柄应该可以单手拖动
    And 底部 "继续拍下一页" 和 "完成采集" 不应该固定悬浮遮挡列表内容

  Scenario: 会话进入修订采集后手机端恢复编辑态
    Given 采集会话曾处于 locked 状态
    And 电脑端已通过修订采集解锁该会话
    When 我刷新手机端采集页
    Then 页面应该恢复为可编辑状态
    And 已上传的页面列表应该保留
    And "拍摄/选择图片" 按钮应该可用
    And "完成采集" 按钮应该可用

  Scenario: 已取消的会话手机端不可访问
    Given 采集会话 "sess_001" 状态为 cancelled
    When 我访问该会话的采集页
    Then 我应该看到 "采集会话已取消" 的提示
    And 所有拍摄/选择、上传、删除、排序、补拍、重新框选按钮都应该处于禁用状态

  Scenario: 已采集列表态按目标设计稿展示短状态和底部操作
    Given 我已成功上传 3 页病历图片
    When 我查看手机采集页
    Then 顶栏标题应为 "已采集页面"
    And 顶部短说明应为 "已采集 3 页，可删除、调整页序或补拍"
    And 页面列表应位于主要内容区域，不应被浏览器底栏或页面底部按钮遮挡
    And 列表下方应显示 "补拍页面" 区域
    And 底部提示条应显示 "请按病历原顺序排列，确保内容完整、清晰、无反光。"
    And 底部应显示 "继续拍下一页" 和 "完成采集" 两个主要操作
```
