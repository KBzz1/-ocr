# 页面列表与页序管理

> 对应 PRD: PR-FE-002 | 依赖: `docs/Shared/state-enums.md`

```gherkin
Feature: 页面列表与页序管理
  作为 采集人
  我想要 管理已采集的页面顺序和内容
  以便 确保提交的病历页面顺序正确

  Scenario: 连续上传多页后页面按采集顺序编号
    Given 我在采集页依次上传了 3 张图片
    When 我查看页面列表
    Then 列表应该显示页序为 1、2、3
    And 每页应显示对应的缩略图
    And 每个已上传页面应显示 "补拍"、"重新框选"、"删除" 行内按钮

  Scenario: 已采集页面按目标设计稿使用横向审阅卡片
    Given 我在采集页依次上传了 3 张图片
    When 我查看 "页面列表"
    Then 每个页面项应形成一行横向卡片
    And 左侧应显示框选后的页面缩略图
    And 中部应显示 "第 1 页"、"已上传" 状态和上传时间
    And 右侧应显示删除操作
    And 页面项应支持长按拖拽排序
    And 页面项不应呈现为大按钮纵向堆叠的凌乱布局

  Scenario: 删除中间页面后剩余页面重新编号
    Given 页面列表中有 3 页 [A, B, C]
    When 我删除第 2 页
    And 我在确认弹窗中点击 "确认删除"
    Then 系统应该调用删除 API
    And 剩余页面应该重新编号为 1 (A)、2 (C)

  Scenario: 取消删除操作后列表保持不变
    Given 页面列表中有 3 页
    When 我点击第 2 页的删除按钮
    And 在确认弹窗中点击 "取消"
    Then 不应该调用删除 API
    And 页面列表应该保持 3 页不变

  Scenario: 拖拽调整页面顺序后调用排序 API
    Given 页面列表为 [A, B, C]
    When 我将 C 拖拽到最前面，顺序变为 [C, A, B]
    Then 系统应该调用排序 API 保存新顺序
    And 页面列表展示新顺序 [C, A, B]

  Scenario: 排序接口失败时恢复原顺序并提示错误
    Given 页面列表为 [A, B, C]
    And 排序 API 将返回 500 错误
    When 我拖拽调整顺序
    Then 页面应该恢复到原来的顺序 [A, B, C]
    And 我应该看到 "排序失败，请重试" 的错误提示

  Scenario: 通过行内按钮补拍页面
    Given 页面列表中有 2 页 [A, B]
    When 我点击第 1 页的 "补拍" 按钮
    And 拍摄一张新图片并上传
    Then 新图片应该替换第 1 页的原图片和框选元数据
    And 第 1 页页序保持不变
    And 页面列表仍为 2 页且不产生重复页序

  Scenario: 补拍区域作为列表后的辅助入口
    Given 页面列表中有 2 页 [A, B]
    When 我查看列表底部
    Then 我应该看到 "补拍页面" 区域
    And 该区域应说明 "若页面模糊、缺失或拍摄不完整，可补拍后替换该页"
    And 点击该区域的 "补拍页面" 应引导用户先选择需要替换的目标页

  Scenario: 上传失败页面只展示重试和删除
    Given 页面列表中第 2 页上传失败
    When 我查看页面列表
    Then 第 2 页应该显示 "上传失败" 状态
    And 第 2 页应该显示 "重试" 和 "删除" 按钮
    And 第 2 页不应该显示 "补拍" 或 "重新框选" 按钮

  Scenario: 锁定后所有编辑入口不可用
    Given 采集会话已锁定
    When 我查看页面列表
    Then 删除按钮应该不可见或禁用
    And 拖拽排序功能应该不可用
    And 补拍按钮应该不可见或禁用
    And 重新框选按钮应该不可见或禁用

  Scenario: 修订采集后页面列表恢复编辑
    Given 采集会话曾被锁定，现已通过电脑端修订采集恢复 active
    When 我刷新手机端采集页
    Then 页面列表应恢复可编辑状态
    And 所有已上传页面应保留
    And 删除、拖拽排序、补拍、重新框选入口应恢复可用
    And "继续拍下一页" 和 "完成采集" 应恢复可用
```
