# 电脑端工作台

> 对应 PRD: PR-FE-001 | 依赖: `docs/Shared/terminology.md`

```gherkin
Feature: 电脑端工作台
  作为 医生
  我想要 在浏览器中查看工作站运行状态
  以便 确认系统已就绪并开始采集病历

  Scenario: 系统正常启动后显示运行状态
    Given 后端服务已正常启动
    When 我访问工作台页面 "http://127.0.0.1:{port}"
    Then 我应该看到 "系统已启动" 的状态提示
    And 我应该看到本地访问地址和端口号
    And 我应该看到 "新建采集" 按钮处于可用状态

  Scenario: 系统状态接口返回异常时展示错误信息
    Given 后端状态接口返回 `status: "error"` 且 `message: "服务启动失败"`
    When 我访问工作台页面
    Then 我应该看到 "服务启动失败" 的错误信息
    And 我不应该看到 "系统已启动" 的状态标识
    And "新建采集" 按钮应该处于禁用状态

  Scenario: 状态接口超时后提供重试入口
    Given 后端状态接口无响应
    When 我访问工作台页面并等待超时
    Then 我应该看到 "服务无响应" 的提示
    And 我应该看到 "重试" 按钮
    When 后端恢复后我点击 "重试"
    Then 页面应该刷新并显示 "系统已启动"

  Scenario: 无病历任务时展示明确空状态提示
    Given 系统中没有任何病历任务
    When 我访问工作台页面
    Then 我应该看到 "暂无病历任务，请使用手机扫码采集" 的提示
    And 我应该看到最近处理记录区域显示 "暂无处理记录"

  Scenario: 成功创建采集会话后展示手机采集二维码
    Given 后端服务正常运行且局域网地址可用
    When 我点击 "新建采集" 按钮
    Then 系统应该调用 POST /api/capture-sessions
    And 我应该看到一个包含采集链接的二维码图片
    And 二维码中的 URL 应该使用局域网 IP 而非 127.0.0.1
    And 我应该看到局域网访问地址的文字说明

  Scenario: 自动识别局域网地址失败时允许用户手动选择
    Given 电脑有多个网卡且自动识别不确定
    When 我访问工作台页面
    Then 我应该看到可手动选择的局域网地址列表
    And 我应该看到可以手动输入/粘贴访问地址的输入框
    When 我选择一个地址后点击 "重新生成二维码"
    Then 二维码中的 URL 应该使用我选择的地址

  Scenario: 创建采集会话失败时不保留旧二维码
    Given 后端的 POST /api/capture-sessions 返回 500 错误
    And 页面上原本没有二维码
    When 我点击 "新建采集" 按钮
    Then 我应该看到 "创建采集会话失败，请重试" 的错误提示
    And 页面上不应该出现二维码
```
