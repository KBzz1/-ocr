# 错误提示与可恢复性

> 对应 PRD: PR-FE-008

```gherkin
Feature: 错误提示与可恢复性
  作为 用户（医生或采集人）
  我想要 在遇到错误时获得清晰提示和恢复路径
  以便 不被错误卡住，能继续完成工作

  Scenario: 手机端无法连接工作站时给出明确指引
    Given 手机与电脑不在同一网络
    When 我通过采集 URL 访问
    Then 我应该看到 "无法连接到工作站，请确认手机与电脑在同一网络"
    And 页面应该提供 "重试" 按钮

  Scenario: 网络恢复后重试成功回到原页面
    Given 手机端因网络问题无法连接工作站
    And 显示了连接错误页面
    When 网络恢复后我点击 "重试"
    Then 应该成功加载采集页面
    And 如果之前有上传记录，应该恢复显示

  Scenario: 处理失败的任务在列表和详情页均可查看原因
    Given 一个任务因算法异常处理失败
    When 我在任务列表中查看
    Then 应该看到失败状态和错误摘要
    When 我进入任务详情页
    Then 也应该能看到完整的错误原因

  Scenario: 算法未配置时不显示人工降级提示
    Given 后端算法模块未配置
    And 一个任务因此失败
    When 我查看该任务
    Then 应该显示失败原因 "算法模块未配置，无法生成结构化字段"
    And 应该显示 "重新处理" 按钮
    And 不应该出现 "人工补录后继续确认/导出" 的操作入口

  Scenario: 字段抽取为空或异常时不允许进入审核页
    Given 任务因 LLM 字段抽取异常而失败
    When 我尝试进入审核页
    Then 应该展示失败态和错误原因
    And 不应该显示可编辑的字段列表
    And "确认审核" 入口不可用

  Scenario: 保存字段失败后允许再次保存
    Given 我在审核页修改了一个字段
    And 保存 API 返回 500 错误
    When 我看到 "保存失败，请重试" 提示
    Then 我可以再次编辑该字段并重新保存
    And 保存按钮应该处于可用状态

  Scenario: 重复操作被按钮忙碌态阻止
    Given 我在审核页
    When 我点击 "确认审核"
    And 请求还未完成
    Then 按钮应该进入 loading 状态
    And 我再次点击 "确认审核" 时不产生第二个请求

  Scenario: 任一流程中刷新页面后状态一致
    Given 任务状态为 "待审核" 且有部分字段已修改
    When 我刷新页面
    Then 任务状态应该仍然是 "待审核"
    And 已修改的字段应该保留修改后的值
    And 页面不应显示不一致或过时的数据
```
