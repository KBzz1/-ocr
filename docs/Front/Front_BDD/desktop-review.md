# 电脑端人工审核

> 对应 PRD: PR-FE-004 | 依赖: `docs/Shared/state-enums.md`

```gherkin
Feature: 电脑端人工审核
  作为 医生
  我想要 在电脑端同时查看原图、文本和结构化字段
  以便 高效核验并修正自动抽取的病历信息

  Scenario: 审核页面展示原图、解析文本和结构化字段三个区域
    Given 任务状态为 "待审核"
    When 我进入该任务的审核页
    Then 页面应该同时显示：
      | 区域         | 说明                 |
      | 原图查看区   | 病历页面原始图片     |
      | 解析文本区   | OCR/文档解析后的文本  |
      | 结构化字段区 | 按字段组展示的字段列表 |

  Scenario: 多页任务通过缩略图切换主图和对应文本
    Given 任务包含 3 页病历图片
    When 我进入审核页
    Then 页面应显示所有页面的缩略图列表
    When 我点击第 2 页的缩略图
    Then 主图区域应该显示第 2 页的原图
    And 解析文本区应该显示或高亮第 2 页对应的文本

  Scenario: 切换查看合并后的全文文本
    Given 任务包含多页
    When 我切换到 "合并文本" 视图
    Then 文本区应该显示所有页面的合并文本
    And 即使合并文本为空，也不应该页面崩溃

  Scenario: 状态为失败的任务无法进入审核
    Given 一个任务状态为 "失败"
    When 我尝试进入该任务的审核页
    Then 我应该看到失败原因和 "重新处理" 入口
    And 我不应该看到可编辑的字段列表
    And 确认审核按钮应该不可用

  Scenario: 点击字段值进入编辑模式
    Given 我在审核页查看到结构化字段
    When 我点击 "主诉" 字段的值
    Then 该字段应该切换为输入框并获得焦点
    And 字段旁应显示 "保存" 和 "取消" 操作

  Scenario: 保存修改后字段状态更新为已修改
    Given 我在审核页编辑 "主诉" 字段的值
    When 我输入新值并点击保存（或失焦）
    Then 系统应该调用 PUT /api/tasks/{taskId}/review/fields/chief_complaint
    And 保存成功后字段状态应变为 "已修改"
    And 字段值应显示我输入的新值

  Scenario: 保存失败时恢复原值并提示错误
    Given 我在审核页编辑 "主诉" 字段的值
    And 保存 API 将返回 500 错误
    When 我输入新值并保存
    Then 字段值应该回滚到修改前的值
    And 我应该看到 "保存失败，请重试" 的错误提示

  Scenario: 清空字段值后状态变为空
    Given 审核页中 "既往史" 字段有自动抽取值
    When 我清空该字段的值并保存
    Then 字段值应该为空
    And 字段状态应该显示为 "为空"

  Scenario: 确认字段后状态变为已确认
    Given 审核页中 "姓名" 字段为 "未审核" 状态
    When 我点击该字段的 "确认" 操作
    Then 字段状态应该变为 "已确认"

  Scenario: 标记存疑后字段显示存疑状态
    Given 审核页中 "初步诊断" 字段有自动抽取值
    When 我点击该字段的 "标记存疑" 操作
    Then 字段状态应该变为 "存疑"
    And 字段应该以视觉方式突出显示存疑状态（如黄色背景）

  Scenario: 页面刷新后加载人工修改后的值
    Given 我已修改 "主诉" 字段并保存
    When 我刷新审核页
    Then "主诉" 字段应该显示我修改后的值
    And 不应显示自动抽取的原始值

  Scenario: 字段保存中时确认审核按钮进入防重复状态
    Given 我在审核页正在保存一个字段
    When 保存请求尚未完成
    And 我点击 "确认审核"
    Then 按钮应该进入防重复提交状态
    And 不应立即发送确认请求
```
