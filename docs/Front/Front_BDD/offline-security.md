# 离线、安全与隐私

> 本节覆盖离线运行、隐私保护和数据安全的前端场景

```gherkin
Feature: 离线、安全与隐私
  作为 系统维护者
  我想要 确保系统完全离线运行且不泄露隐私数据
  以便 满足院内数据安全要求

  Scenario: 页面加载时不请求任何外部资源
    Given 系统部署在离线 Windows 电脑上
    When 我打开工作台页面
    Then 页面不应该请求任何 CDN 资源
    And 不应该请求任何远程字体
    And 不应该请求任何外部域名
    And 所有静态资源应来自本地服务

  Scenario: 浏览器离线状态下核心流程不受影响
    Given 浏览器已加载页面且处于离线状态
    And 后端服务在本地运行
    When 我进行任务列表查看、审核、导出操作
    Then 所有操作应该正常完成
    And 不应因无法加载外部资源而报错

  Scenario: 错误 toast 不展示完整病历原文和身份证号
    Given 后端返回错误时附带了一些上下文信息
    When 前端展示错误提示
    Then toast 中不应包含完整的病历文本原文
    And 不应包含身份证号
    And 不应包含手机号等个人隐私信息

  Scenario: 含路径穿越字符的文件名只展示安全名称
    Given 后端返回的文件名包含 "../" 路径穿越字符
    When 前端展示该文件名
    Then 应该只展示净化后的安全文件名
    And 不应将路径拼接到本地文件系统路径中

  Scenario: 浏览器控制台不输出敏感信息
    Given 审核页展示了病历的 OCR 文本和字段内容
    When 我查看浏览器控制台
    Then 控制台不应输出完整的 OCR 识别文本
    And 不应输出结构化字段的原文
    And 不应输出图片的 base64 数据

  Scenario: 测试环境中未 mock 的请求必须让测试失败
    Given 测试环境使用 MSW 进行 API mock
    When 前端代码发起了一个未被 mock 的 API 请求
    Then 测试必须失败
    And 失败原因应提示哪个请求未被 mock
```
