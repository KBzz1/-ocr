# 前端 TDD — 手机端采集会话

> PRD: PR-FE-002
> 关联: `05-page-management.md`
> 会话状态见 `docs/Shared/state-enums.md`

| ID | 类型 | 用例 |
|----|------|------|
| FE-CAP-001 | 组件 | URL 含 `?session=sess_001` 时，请求会话详情并显示已采集页数 |
| FE-CAP-002 | 组件 | 会话不存在时显示"无效的采集链接，请重新扫描二维码"，禁用拍摄和上传 |
| FE-CAP-003 | 组件 | 会话过期时显示"采集会话已过期"，禁用所有编辑动作 |
| FE-CAP-004 | 组件 | 会话 `locked` 后进入页面，显示"采集已完成，请在电脑端查看"，禁止新增、删除、排序、补拍、重新框选 |
| FE-CAP-005 | 组件 | 点击单个"拍摄/选择图片"触发文件输入或相机入口，选取图片后进入框选页并显示本地预览 |
| FE-CAP-006 | 组件 | 单个文件入口可选取 jpg/png/bmp 图片并显示预览 |
| FE-CAP-007 | 组件 | 选择非图片文件时前端拦截，显示"不支持的文件类型"，不调用上传 API |
| FE-CAP-008 | 组件 | 图片超过限制时前端拦截，显示"图片过大（最大 20MB）"，不调用上传 API |
| FE-CAP-009 | 组件 | 上传中显示该页 loading/进度状态，重复点击上传按钮不产生重复请求 |
| FE-CAP-010 | 组件 | 上传失败保留本地图片和页序，显示"上传失败，请重试"与重试按钮 |
| FE-CAP-011 | 组件 | 点击重试后只重新上传失败页，不重复上传已成功页面 |
| FE-CAP-012 | 组件 | 未采集任何页面点击"完成采集"时提示"请至少采集一页病历"，不调用 finish API |
| FE-CAP-013 | 组件 | 已上传页面后点击"完成采集"，调用 `POST /api/mobile/{sessionId}/finish` |
| FE-CAP-014 | 组件 | 完成采集成功后页面切换为只读完成态，提示到电脑端查看 |
| FE-CAP-015 | 组件 | 上传成功后返回页面列表，已上传页显示缩略图、页序、"已上传"状态和补拍/重新框选/删除行内按钮 |
| FE-CAP-016 | 组件 | 上传失败页只显示重试/删除按钮，不显示补拍/重新框选 |
| FE-CAP-017 | E2E | 移动端视口下主要按钮不遮挡、不溢出，单手操作区域可点击，底部操作不使用 fixed 遮挡列表 |
