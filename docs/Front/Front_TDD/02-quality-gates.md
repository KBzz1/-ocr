# 前端 TDD — 质量门禁

每个用例落地时必须满足：

- 测试名描述一个可观察行为，避免 `works`、`test1` 之类命名。
- 先写测试并运行到 RED，失败原因应是功能缺失或断言不满足，而不是 import/语法错误。
- mock 数据必须是最小必要数据，不通过大而全 fixture 掩盖断言。
- 对异步 UI 使用 `findBy*`、`waitFor` 或 Playwright expect，不使用固定 `sleep`。
- 所有外部请求、CDN 字体、远程图片请求必须在测试中失败。
- 算法相关用例只测"外部成功返回/错误提示"，不测实际算法结果；算法未配置、异常、结构化字段为空或契约非法时必须按失败处理。
