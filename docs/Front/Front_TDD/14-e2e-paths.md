# 前端 TDD — 端到端关键路径

## FE-E2E-001：完整采集到人工导出

1. 电脑端打开工作台，显示系统状态和空任务列表。
2. 新建采集会话，任务列表新增 `capturing` 任务；弹窗显示二维码、会话状态、剩余时间（默认 30 分钟）和已上传页数；首页不直接展示 IP、端口或完整采集地址。
3. 移动端通过 session URL 进入采集页。
4. 选择 3 张本地 fixture 图片。
5. 每页手动调整四边形角点，只断言坐标变化和上传请求体，不断言图像处理效果。
6. 完成采集后会话锁定，任务变为 `uploaded`。
7. 电脑端任务列表出现任务，状态流转到 `ready_for_review`。
8. 打开审核页，展示后端成功返回的解析文本和结构化字段候选。
9. 修改字段后失焦自动保存；批量勾选字段并一键确认；标记一个字段存疑；确认一个空值字段为"空值可接受"。
10. 确认审核：因存疑字段弹出预警窗，选择继续确认，任务进入 `confirmed`。
11. 导出前展示完整性检查预警面板，选择继续导出后触发 JSON 和 Excel 下载，任务变为 `exported`。

## FE-E2E-002：算法模块失败必须报错

后端返回 `ALGORITHM_MODULE_NOT_CONFIGURED` 时：

- 任务状态为 `failed`。
- 任务列表和详情页显示错误原因。
- 审核页入口不可用或进入后只显示失败态。
- 页面提供重新处理入口。
- 不出现"人工补录后继续确认/导出"的降级路径。

## FE-E2E-003：离线约束

- 页面不请求外部域名。
- 所有 mock API 使用本地地址。
- 断网不导致前端加载远程资源失败。

## FE-E2E-004：修订采集重新编辑

1. 完成采集后任务状态为 `uploaded`。
2. 电脑端点击"修订采集"并确认。
3. 任务回到 `capturing`，手机端恢复可编辑。
4. 手机端补拍第 2 页、拖拽调整页序。
5. 重新完成采集，任务再次进入 `uploaded → processing → ready_for_review`。
6. 会话 ID 和任务 ID 保持不变。

## FE-E2E-005：取消采集

1. 创建采集会话后任务状态为 `capturing`。
2. 已上传 2 页。
3. 电脑端点击"取消采集"并确认。
4. 会话进入 `cancelled`，任务进入 `failed`。
5. 手机端显示"采集会话已取消"。
