# 前端 TDD — 端到端关键路径

## FE-E2E-001：完整采集到人工导出

1. 电脑端打开工作台，显示系统状态和空任务列表。
2. 新建采集会话，弹窗显示二维码、会话状态、剩余时间和已上传页数；首页不直接展示 IP、端口或完整采集地址。
3. 移动端通过 session URL 进入采集页。
4. 选择 3 张本地 fixture 图片。
5. 每页调整四边形角点，只断言坐标变化和上传请求体，不断言图像处理效果。
6. 完成采集后会话锁定。
7. 电脑端任务列表出现新任务，状态流转到 `ready_for_review`。
8. 打开审核页，展示后端成功返回的解析文本和结构化字段候选。
9. 修改字段，确认若干字段，标记一个字段存疑。
10. 确认审核先因存疑字段被阻断；处理存疑并确认全部字段后，任务进入 `confirmed`。
11. 导出 JSON 和 Excel，验证触发下载和导出记录。

## FE-E2E-002：算法模块失败必须报错

后端返回 `ALGORITHM_MODULE_NOT_CONFIGURED` 时：

- 任务状态为 `failed`。
- 任务列表和详情页显示错误原因。
- 审核页入口不可用或进入后只显示失败态。
- 页面提供重新处理入口。
- 不出现"人工补录后继续确认/导出"的降级路径。

## FE-E2E-003：离线约束

- 页面不请求外部域名。
- 所有 mock API 使用本地地址。
- 断网不导致前端加载远程资源失败。
