# 前端 TDD — 电脑端工作台

> PRD: PR-FE-001
> 状态定义见 `docs/Shared/state-enums.md`

| ID | 类型 | 用例 |
|----|------|------|
| FE-WKS-001 | 组件 | `GET /api/system/status` 返回 `running` 时，首页显示"系统已启动"、"离线运行"、"手机采集可用"，不直接显示 IP 或端口 |
| FE-WKS-002 | 组件 | 状态接口返回 `error` 时，展示后端错误消息，不渲染误导性的 running 状态 |
| FE-WKS-003 | 组件 | 状态接口超时后显示"服务无响应"，并提供重试按钮 |
| FE-WKS-004 | 组件 | 任务列表为空时显示"暂无病历任务，请使用手机扫码采集" |
| FE-WKS-005 | 组件 | 最近处理记录为空时显示"暂无处理记录" |
| FE-WKS-006 | 组件 | 点击"新建采集"后调用 `POST /api/capture-sessions`，同时创建 `capturing` 任务；成功后显示二维码、会话状态、剩余时间（默认 30 分钟）和已上传页数；弹窗中可修改过期时间 |
| FE-WKS-007 | 组件 | 二维码 URL 使用局域网 IP，不使用 `127.0.0.1` 作为手机访问地址；二维码内容不以明文常驻首页 |
| FE-WKS-008 | 组件 | 多网卡或自动识别失败时，只在"手机无法连接？"高级说明中允许用户手动选择/复制访问地址并重新生成二维码 |
| FE-WKS-009 | 组件 | 创建采集会话失败时显示"创建采集会话失败，请重试"，且不保留旧二维码 |
| FE-WKS-010 | E2E | 浏览器访问 `http://127.0.0.1:{port}` 后页面可加载，无 console error |
| FE-WKS-011 | 组件 | 任务统计颜色和中文文案与 `docs/Shared/state-enums.md` 对齐：`capturing` 采集中、`ready_for_review` 待审核、`processing` 处理中、`failed` 失败、`exported` 已导出 |
| FE-WKS-012 | 组件 | 首页提醒区标题为"系统提醒"，失败任务操作文案为"查看原因"或"重新处理"，不向医生展示"查看日志" |
| FE-WKS-013 | 组件 | 从首页侧边栏进入 `/tasks` 后仍保留工作站侧边栏，提供可返回 `/` 的"工作台总览"入口，并激活"任务管理"导航态 |
| FE-WKS-014 | 组件 | 当前采集会话卡片展示已上传页面缩略图（横向滚动），缩略图基于 quad_points 生成，点击可放大查看 |
| FE-WKS-015 | 组件 | 点击当前会话卡片的"取消采集"并确认后，调用取消 API，会话变为 `cancelled`，任务变为 `failed` |
| FE-WKS-016 | 组件 | 已锁定会话卡片在任务不为 `processing` 时显示"修订采集"按钮，确认后调用解锁 API，会话回到 `active`，任务回到 `capturing` |
| FE-WKS-017 | 组件 | 任务为 `processing` 时"修订采集"禁用，并显示当前状态不可修订 |
| FE-WKS-018 | 组件 | 最近任务列表每项展示首页缩略图，`capturing` 状态无页面时显示占位图 |

关键 RED 断言：

- 如果组件把 `127.0.0.1` 放进二维码，FE-WKS-007 必须失败。
- 如果首页常驻展示 IP、端口或完整采集 URL，FE-WKS-001 必须失败。
- 如果手动选择局域网地址后二维码不更新，FE-WKS-008 必须失败。
- 如果 `/tasks` 渲染为缺少工作站侧边栏的孤立页面，FE-WKS-013 必须失败。
- 如果页面加载远程 CSS、图标、字体或图片资源，FE-SEC-001 必须失败。
- 如果 `capturing` 状态显示为旧名"已创建"，FE-WKS-011 必须失败。
- 如果任务处理中时"修订采集"可点击，FE-WKS-017 必须失败。
