# 前端 TDD — 电脑端工作台

> PRD: PR-FE-001
> 状态定义见 `docs/Shared/state-enums.md`

| ID | 类型 | 用例 |
|----|------|------|
| FE-WKS-001 | 组件 | `GET /api/system/status` 返回 `running` 时，显示"系统已启动"、本地地址和端口 |
| FE-WKS-002 | 组件 | 状态接口返回 `error` 时，展示后端错误消息，不渲染误导性的 running 状态 |
| FE-WKS-003 | 组件 | 状态接口超时后显示"服务无响应"，并提供重试按钮 |
| FE-WKS-004 | 组件 | 任务列表为空时显示"暂无病历任务，请使用手机扫码采集" |
| FE-WKS-005 | 组件 | 最近处理记录为空时显示"暂无处理记录" |
| FE-WKS-006 | 组件 | 点击"新建采集"后调用 `POST /api/capture-sessions`，成功后显示二维码 |
| FE-WKS-007 | 组件 | 二维码 URL 使用局域网 IP，不使用 `127.0.0.1` 作为手机访问地址 |
| FE-WKS-008 | 组件 | 多网卡或自动识别失败时，允许用户手动选择/复制访问地址并重新生成二维码 |
| FE-WKS-009 | 组件 | 创建采集会话失败时显示"创建采集会话失败，请重试"，且不保留旧二维码 |
| FE-WKS-010 | E2E | 浏览器访问 `http://127.0.0.1:{port}` 后页面可加载，无 console error |

关键 RED 断言：

- 如果组件把 `127.0.0.1` 放进二维码，FE-WKS-007 必须失败。
- 如果手动选择局域网地址后二维码不更新，FE-WKS-008 必须失败。
