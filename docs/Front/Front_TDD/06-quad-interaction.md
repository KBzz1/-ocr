# 前端 TDD — 四边形框选交互

> PRD: PR-FE-009
> 只测前端交互和坐标契约，不测自动识别边界、裁剪、透视矫正、去噪等图像处理效果。

| ID | 类型 | 用例 |
|----|------|------|
| FE-QUAD-001 | 组件 | 图片预览后显示四边形 overlay 和四个角点手柄 |
| FE-QUAD-002 | 单元 | 默认角点按图片尺寸计算，约覆盖中心 80% 区域 |
| FE-QUAD-003 | 组件 | 拖动左上角点后，状态中的 `quad_points.tl` 更新并重绘 overlay |
| FE-QUAD-004 | 组件 | 拖动任一角点不得超出图片边界 |
| FE-QUAD-005 | 单元 | 自相交四边形被判定为无效，UI 不提交该坐标 |
| FE-QUAD-006 | 组件 | 用户未调整时可直接确认，上传默认 `quad_points` |
| FE-QUAD-007 | 组件 | 自动边界结果若由外部模块返回，前端可展示；用户手动拖动后以手动坐标为准 |
| FE-QUAD-008 | 组件 | 外部图像预览模块未配置时，显示"裁剪预览暂不可用"，不伪造矫正预览 |
| FE-QUAD-009 | 组件 | 确认上传时请求体包含原图文件、`quad_points`、页序、图片尺寸 |
| FE-QUAD-010 | 组件 | overlay 抛错时由错误边界接管，提供"重新拍摄"入口，不生成假处理图 |

禁止断言：

- 禁止断言裁剪后图片像素内容。
- 禁止断言摩尔纹、反光、倾斜被修正。
- 禁止断言自动识别边界准确。
