# 前端 TDD — 四边形框选交互

> PRD: PR-FE-009
> 只测前端交互和坐标契约。框选完全由用户手动调节，本项目不实现自动边界识别算法。

| ID | 类型 | 用例 |
|----|------|------|
| FE-QUAD-001 | 组件 | 图片预览后显示四边形 overlay 和四个角点手柄 |
| FE-QUAD-002 | 单元 | 默认角点按图片尺寸计算，约覆盖中心 80% 区域 |
| FE-QUAD-003 | 组件 | 拖动左上角点后，状态中的 `quad_points.tl` 更新并重绘 overlay |
| FE-QUAD-004 | 组件 | 拖动任一角点不得超出图片边界 |
| FE-QUAD-005 | 单元 | 自相交四边形被判定为无效，UI 不提交该坐标 |
| FE-QUAD-006 | 组件 | 用户未调整时可直接确认，上传默认 `quad_points` |
| FE-QUAD-007 | 组件 | 确认上传时请求体包含原图文件、`quad_points`、页序、图片尺寸 |
| FE-QUAD-008 | 组件 | overlay 抛错时由错误边界接管，提供"重新拍摄"入口，不生成假处理图 |
| FE-QUAD-009 | 组件 | 框选页不渲染 X/Y 像素值输入、坐标数值面板或滑块 |
| FE-QUAD-010 | 组件 | 点击已上传页"重新框选"后以该页图片和已保存 `quad_points` 打开框选页 |
| FE-QUAD-011 | 组件 | 已上传页重新框选确认后调用 `PUT /api/mobile/{sessionId}/pages/{pageId}/quad`，不重新上传原图 |
| FE-QUAD-012 | 组件 | `locked` 或 `expired` 或 `cancelled` 会话中重新框选入口不可用 |
| FE-QUAD-013 | 组件 | pointer down 发生在图片空白区域或四边形边线上时，不选择最近角点、不移动任何角点 |
| FE-QUAD-014 | 组件 | pointer down 发生在某个圆点手柄上时，只激活该角点；pointer move 期间其他三个角点保持不变 |
| FE-QUAD-015 | 组件 | 快速拖动角点时使用 pointer capture 保持连续拖动，overlay 不丢失拖动状态 |
| FE-QUAD-016 | 单元 | 四角坐标按左上、右上、右下、左下固定顺序输出为 `quad_points`，拖动后不重排角点语义 |
| FE-QUAD-017 | 组件 | 拖动造成自相交后点击确认上传或确认框选，提交被阻止并显示"框选区域无效，请重新调整" |
| FE-QUAD-018 | 组件 | 重新框选已上传页后点击返回或取消，不调用 `PUT quad`，页面列表仍使用原已保存坐标 |
| FE-QUAD-019 | E2E | 手机触控模拟拖动圆点时，圆点随手指移动且四边形不出现交叉线、三角形或角点跳变 |
| FE-QUAD-020 | 组件 | 用户手动调整后的坐标以上传时确认为准，不展示自动边界识别结果 |

禁止断言：

- 禁止断言裁剪后图片像素内容。
- 禁止断言摩尔纹、反光、倾斜被修正。
- 禁止断言自动识别边界准确（本项目不实现）。
- 禁止断言前端根据图片内容自动生成框选结果；默认范围只能按图片尺寸计算。
