# 前端 TDD — 审查结论与测试总原则

> 基于 `docs/产品PRD.md` 的 PR-FE-001 ~ PR-FE-009 编写。
> 共享定义见 `docs/Shared/state-enums.md`、`docs/Shared/error-codes.md`。

## 审查结论与边界修正

- 图像处理效果（裁剪、透视矫正、去摩尔纹等）交给外部团队，前端只测交互、坐标采集和上传契约。
- 不测试 OCR/LLM 语义正确性。前端只展示后端成功返回的解析文本、字段候选、来源证据和字段状态。
- 状态枚举补齐 `created`、`uploading` 等 PRD 状态；算法模块未配置、异常或结构化抽取失败时必须显示 `failed`，不得进入人工降级流程。
- 多页采集必须覆盖"完成采集前可编辑，完成后锁定"。
- 缺少手动选择局域网地址、无外部网络请求、隐私日志不外泄、未 mock 请求失败等质量门禁。

---

## 本项目前端负责

- 本地工作台展示、二维码与采集入口展示
- 手机端拍照/选择图片、四边形框选交互、页序管理、上传状态
- 任务列表、状态流转展示、失败重试入口
- 审核页原图/文本/字段展示、字段编辑、状态管理、来源核验
- 导出触发、导出前校验、错误提示和恢复路径

## 本项目前端不负责

- OCR 文字识别准确率
- LLM 字段抽取准确率
- 自动识别文档边界的算法正确性
- 透视矫正、裁剪、去摩尔纹、去反光等图像处理效果
- 医学诊断、医学建议或字段医学合理性判断

涉及上述能力时，前端测试只验证：

- 是否调用本地后端 API 或注入的外部模块适配器
- 外部模块不可用、异常或返回无效结果时是否显示明确错误
- 后端返回成功结果中的空字段、无来源字段时页面是否可审核；但算法失败或未配置时不得进入审核页
