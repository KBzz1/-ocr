# 前端 TDD — 错误提示与可恢复性

> PRD: PR-FE-008

| ID | 类型 | 用例 |
|----|------|------|
| FE-ERR-001 | 组件 | 手机无法连接电脑时显示"无法连接到工作站，请确认手机与电脑在同一网络" |
| FE-ERR-002 | 组件 | 连接错误页面提供重试按钮，重试成功后恢复原页面 |
| FE-ERR-003 | 组件 | 任务处理失败后，任务列表和详情页均可查看错误原因 |
| FE-ERR-004 | 组件 | 算法模块未配置时显示失败原因和重新处理入口，不显示人工降级提示 |
| FE-ERR-005 | 组件 | LLM 字段抽取为空、异常或契约非法时显示失败态，不允许打开审核页继续确认 |
| FE-ERR-006 | 组件 | 保存审核结果失败后允许再次保存 |
| FE-ERR-007 | 组件 | 重复提交、重复上传、重复导出被前端按钮忙碌态阻止 |
| FE-ERR-008 | E2E | 任一流程失败后刷新页面，可从后端状态恢复到一致 UI |
| FE-ERR-009 | 组件 | 取消采集会话后任务进入 `failed`，错误原因显示"用户取消采集" |
| FE-ERR-010 | 组件 | 修订采集后任务回到 `capturing`，手机端恢复可编辑，会话 ID 和任务 ID 保持不变 |
| FE-ERR-011 | 组件 | 修订后重新完成采集，任务依次经过 `uploaded` → `processing` → `ready_for_review` |
| FE-ERR-012 | 组件 | 会话被取消后手机端显示"采集会话已取消"，所有操作禁用 |
