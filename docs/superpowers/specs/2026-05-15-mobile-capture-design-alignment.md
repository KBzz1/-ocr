# 手机采集页目标设计稿对齐修正设计

## 背景

当前真实手机采集页与 `docs/UI_image/目标设计稿/` 的未拍照态、框选态、已采集列表态存在明显偏差：页面文案过多，框选页需要下滑才能找到关键操作，四角拖动会因点击空白区域导致角点瞬移和四边形交叉，列表没有稳定展示框选后的页面预览，页面列表排序不可用或不易用，已采集列表布局呈纵向大按钮堆叠，不利于审阅。

本设计只修正手机端采集体验和测试契约，不改变后端算法边界。前端仍只上传原图、图片尺寸、页序和用户确认的 `quad_points`，不实现 OCR、透视矫正、图像增强、自动边界识别或字段推断。

## 目标设计稿来源

- `docs/UI_image/目标设计稿/手机端未拍照前的上半部分ui.png`
- `docs/UI_image/目标设计稿/手机端拍照时ui.png`
- `docs/UI_image/目标设计稿/手机端完成拍照后的下半部分ui.png`

这些图片作为视觉和交互逻辑基准，不要求像素级复刻，但必须遵守它们表达的信息层级、操作位置和状态规则。

## 范围

- 未拍照态：顶栏、绿色状态胶囊、一行短说明、浅蓝采集卡、已采集页数、单个"拍摄/选择图片"入口、空列表、禁用完成采集。
- 框选态：顶栏返回必须可回主页面；浅蓝提示条、图片预览、四边形 overlay、坐标上传说明和底部三个操作在 360px~428px 主流移动宽度内关键可见。
- 四边形交互：只允许从圆点手柄开始拖动；点击图片空白处或边线不得让最近角点瞬移；拖动期间只移动激活角点；角点语义固定为左上、右上、右下、左下；自相交不得提交。
- 已采集列表态：列表使用横向审阅卡片；左侧展示用户确认框选后的四边形区域预览；中部展示页序、上传状态、上传时间；右侧提供删除操作和拖拽排序手柄。
- 排序：仅支持拖拽排序，不提供上移/下移按钮；排序失败回滚。
- 补拍：列表底部展示补拍辅助区域；指定页面补拍仍通过页面项操作进入，保持 `page_id/page_no` 不变。
- 文档和测试：同步更新 PRD、PRD 任务清单、Front BDD、Front TDD，并补充本 spec。

## 非范围

- 不新增图像处理算法，不做透视矫正、裁剪落盘、去噪、去摩尔纹或 OCR。
- 不从图片、schema、OCR 文本或页面内容推断结构化字段。
- 不修改电脑端审核、导出、后端任务状态机或算法端口。
- 不要求前端刷新后一定能恢复真实图片缩略图；若后端未提供可访问图片 URL，可显示通用占位，但新上传的本地会话内必须显示框选区域预览。

## 页面状态设计

### 未拍照态

标题为“病历文书采集”。状态区左侧为绿色“采集会话进行中”胶囊，右侧只保留一行说明“请拍摄病历文书页面，完成后回到电脑端审核”。主卡片呈浅蓝弱背景，展示“请拍摄病历文书页面”、已采集 0 页、本地图形或插画，并提供单个“拍摄/选择图片”入口。页面列表显示“暂未上传页面”，完成采集按钮禁用，并提示“至少上传 1 页后可完成采集”。

### 框选态

标题为“调整识别范围”。顶栏左侧返回按钮不调用浏览器历史退出采集页，而是回到进入框选前的主页面状态。新拍图片返回未拍照态或已采集列表态；重新框选返回已采集列表态，并且不调用上传或更新框选 API。

框选页顶部显示浅蓝提示条“请框选病历正文区域，排除屏幕边缘、灰色背景和工具栏”。图片预览区域尽量占据主体空间，底部显示“将上传原图与框选坐标”，操作按钮为“重拍”“重新框选”“确认上传”。在 360px~428px 主流移动宽度下，顶栏、提示条、图片主体、说明和三个按钮应无需下滑即可看到关键内容。

### 已采集列表态

标题为“已采集页面”。状态说明为“已采集 N 页，可删除、调整页序或补拍”。页面列表卡片横向排布：左侧缩略图，中部页序、绿色“已上传”状态和上传时间，右侧删除与拖拽排序手柄。列表后显示补拍页面区域和提示条“请按病历原顺序排列，确保内容完整、清晰、无反光。”底部显示“继续拍下一页”和“完成采集”。

## 四边形预览

上传成功后，页面列表缩略图应优先展示用户框选后的四边形区域预览。前端可以使用 CSS `clip-path: polygon(...)`、SVG mask 或 canvas 生成本地预览，但只能基于用户确认的 `quad_points` 和原图预览 URL，不做透视矫正或内容识别。若页面来自后端历史数据且没有可访问图片 URL，显示占位缩略图并保留页序、状态和操作。

## 验收标准

- 真实手机未拍照态不再出现多段长说明；保留单个“拍摄/选择图片”入口。
- 框选页左上返回能回主页面，取消重新框选不修改已保存页面。
- 四边形拖动连续跟手，点击空白区域不移动角点，不出现交叉线或三角形。
- 新上传页面在列表中展示框选区域预览。
- 页面列表为横向审阅布局，排序仅通过拖拽完成，不显示上移/下移按钮。
- 360px~428px 主流移动宽度下框选页关键内容无需下滑即可操作。
- 文档、BDD、TDD 与 PRD 不要求本仓库实现算法处理。

## 自审

- Placeholder scan：无 TBD、TODO 或待定项。
- Internal consistency：PRD、BDD、TDD 和本 spec 均保持“只收集原图与四边形坐标，不做算法裁剪/矫正”的边界。
- Scope check：范围集中在手机采集页 UX、交互和测试契约，可由一个实施计划拆分为文档、框选组件、列表组件、样式和测试任务。
- Ambiguity check：返回行为、空白点击不瞬移、列表预览、仅拖拽排序和多宽度单屏关键可见性均已写成可验收规则。
