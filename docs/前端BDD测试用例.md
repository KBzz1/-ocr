# 病历文书结构化采集与人工核验工作站 — 前端 BDD 测试用例

> 基于 `docs/产品prd.md` 的 PR-FE-001 ~ PR-FE-009 编写。本文档使用 Gherkin 语法（Given-When-Then）描述前端行为驱动测试场景，聚焦用户可见的业务行为，与 `前端TDD测试用例.md` 互补。

---

## 1. BDD 测试总原则

### 1.1 适用范围

本文档覆盖端到端用户旅程和关键业务场景，包括：

- 电脑端工作台的访问与状态展示
- 手机端采集会话的完整流程
- 任务列表的状态流转与筛选
- 电脑端人工审核的字段编辑与确认
- 字段来源核验交互
- 导出功能与完整性校验
- 错误场景与可恢复性
- 算法模块失败态的前端表现

### 1.2 不覆盖

- 算法效果（OCR 准确率、LLM 抽取质量、图像处理效果）
- 后端内部逻辑（仅通过 API 契约验证）
- 性能与压力测试

### 1.3 Feature 文件组织

```
features/
├── workstation.feature          # 电脑端工作台
├── mobile-capture.feature       # 手机端文档采集
├── page-management.feature      # 页面列表与页序管理
├── quad-selection.feature       # 四边形框选交互
├── task-list.feature            # 任务列表
├── desktop-review.feature       # 电脑端人工审核
├── field-evidence.feature       # 字段来源核验
├── field-status.feature         # 字段状态管理
├── export.feature               # 导出功能
├── error-recovery.feature       # 错误提示与可恢复性
└── offline-security.feature     # 离线、安全与隐私
```

## 2. 测试环境与术语

| 术语 | 说明 |
|------|------|
| 医生 | 电脑端操作用户，负责审核、确认和导出 |
| 采集人 | 手机端操作用户，负责拍照和上传（通常与医生为同一人） |
| 工作站 | 运行在本地 Windows 电脑上的后端服务 + 前端页面 |
| 采集二维码 | 电脑端生成的二维码，手机扫码后进入采集页 |
| 四边形框选 | 手机端拍摄后用于标记病历有效区域的四边形 overlay |
| 结构化字段 | 从病历文书抽取出的字段名-字段值对，如"主诉：头痛3天" |

### 2.1 状态枚举

**任务状态：**

| 后端值 | 前端展示标签 |
|--------|-------------|
| `created` | 已创建 |
| `uploading` | 上传中 |
| `uploaded` | 上传完成 |
| `processing` | 处理中 |
| `ready_for_review` | 待审核 |
| `confirmed` | 已确认 |
| `exported` | 已导出 |
| `failed` | 失败 |

**字段状态：**

| 后端值 | 前端展示标签 |
|--------|-------------|
| `unreviewed` | 未审核 |
| `confirmed` | 已确认 |
| `modified` | 已修改 |
| `suspicious` | 存疑 |
| `empty` | 为空 |

## 3. Feature：电脑端工作台（PR-FE-001）

### 场景：系统正常启动后显示运行状态

```gherkin
Feature: 电脑端工作台
  作为 医生
  我想要 在浏览器中查看工作站运行状态
  以便 确认系统已就绪并开始采集病历

  Scenario: 系统正常启动后显示运行状态
    Given 后端服务已正常启动
    When 我访问工作台页面 "http://127.0.0.1:{port}"
    Then 我应该看到 "系统已启动" 的状态提示
    And 我应该看到本地访问地址和端口号
    And 我应该看到 "新建采集" 按钮处于可用状态
```

### 场景：系统启动异常时展示错误

```gherkin
  Scenario: 系统状态接口返回异常时展示错误信息
    Given 后端状态接口返回 `status: "error"` 且 `message: "服务启动失败"`
    When 我访问工作台页面
    Then 我应该看到 "服务启动失败" 的错误信息
    And 我不应该看到 "系统已启动" 的状态标识
    And "新建采集" 按钮应该处于禁用状态
```

### 场景：服务无响应时展示重试入口

```gherkin
  Scenario: 状态接口超时后提供重试入口
    Given 后端状态接口无响应
    When 我访问工作台页面并等待超时
    Then 我应该看到 "服务无响应" 的提示
    And 我应该看到 "重试" 按钮
    When 后端恢复后我点击 "重试"
    Then 页面应该刷新并显示 "系统已启动"
```

### 场景：任务列表为空时展示空状态

```gherkin
  Scenario: 无病历任务时展示明确空状态提示
    Given 系统中没有任何病历任务
    When 我访问工作台页面
    Then 我应该看到 "暂无病历任务，请使用手机扫码采集" 的提示
    And 我应该看到最近处理记录区域显示 "暂无处理记录"
```

### 场景：创建采集会话并生成二维码

```gherkin
  Scenario: 成功创建采集会话后展示手机采集二维码
    Given 后端服务正常运行且局域网地址可用
    When 我点击 "新建采集" 按钮
    Then 系统应该调用 POST /api/capture-sessions
    And 我应该看到一个包含采集链接的二维码图片
    And 二维码中的 URL 应该使用局域网 IP 而非 127.0.0.1
    And 我应该看到局域网访问地址的文字说明
```

### 场景：多网卡时用户可手动选择地址

```gherkin
  Scenario: 自动识别局域网地址失败时允许用户手动选择
    Given 电脑有多个网卡且自动识别不确定
    When 我访问工作台页面
    Then 我应该看到可手动选择的局域网地址列表
    And 我应该看到可以手动输入/粘贴访问地址的输入框
    When 我选择一个地址后点击 "重新生成二维码"
    Then 二维码中的 URL 应该使用我选择的地址
```

### 场景：创建采集会话失败时展示错误

```gherkin
  Scenario: 创建采集会话失败时不保留旧二维码
    Given 后端的 POST /api/capture-sessions 返回 500 错误
    And 页面上原本没有二维码
    When 我点击 "新建采集" 按钮
    Then 我应该看到 "创建采集会话失败，请重试" 的错误提示
    And 页面上不应该出现二维码
```

## 4. Feature：手机端文档采集（PR-FE-002）

### 场景：通过有效会话链接进入采集页

```gherkin
Feature: 手机端文档采集
  作为 采集人
  我想要 用手机拍摄病历文书页面并上传到工作站
  以便 在电脑端进行结构化审核

  Scenario: 通过有效会话链接进入采集页
    Given 存在一个有效的采集会话 "sess_001"
    When 我通过 URL "http://192.168.1.100:{port}/capture?session=sess_001" 访问采集页
    Then 我应该看到采集页面
    And 我应该看到 "已采集 0 页" 的提示
    And 我应该看到 "拍照" 和 "选择已有图片" 按钮
```

### 场景：无效会话链接时展示错误

```gherkin
  Scenario: 会话不存在时禁用采集功能
    Given 会话 "sess_invalid" 不存在
    When 我通过 URL 访问 "?session=sess_invalid"
    Then 我应该看到 "无效的采集链接，请重新扫描二维码"
    And "拍照" 按钮应该处于禁用状态
    And "选择已有图片" 按钮应该处于禁用状态
```

### 场景：会话过期时禁用所有操作

```gherkin
  Scenario: 会话过期后禁止新增内容
    Given 采集会话 "sess_001" 已过期
    When 我访问该会话的采集页
    Then 我应该看到 "采集会话已过期" 的提示
    And 所有拍摄、选择、上传、删除、排序按钮都应该处于禁用状态
```

### 场景：已完成采集的会话显示只读状态

```gherkin
  Scenario: 已锁定的会话禁止编辑
    Given 采集会话 "sess_001" 状态为 locked
    When 我访问该会话的采集页
    Then 我应该看到 "采集已完成，请在电脑端查看" 的提示
    And 拍照、删除、排序、补拍入口均不可用
```

### 场景：拍照并预览

```gherkin
  Scenario: 拍照后显示本地预览
    Given 我在采集页且会话有效
    When 我点击 "拍照" 按钮
    Then 应该触发设备的相机或文件选择
    When 我选取一张图片
    Then 我应该看到该图片的本地预览
    And 我应该看到四边形框选 overlay 叠加在图片上
```

### 场景：选择已有图片

```gherkin
  Scenario: 选择相册中的图片作为病历页面
    Given 我在采集页且会话有效
    When 我点击 "选择已有图片"
    And 我选择一张 jpg 图片
    Then 我应该看到该图片的预览
    And 我应该看到四边形框选 overlay
```

### 场景：选择非图片文件时前端拦截

```gherkin
  Scenario: 选择 PDF 文件时前端直接拦截
    Given 我在采集页且会话有效
    When 我点击 "选择已有图片" 并选择一个 pdf 文件
    Then 我应该看到 "不支持的文件类型" 的错误提示
    And 不应该调用上传 API
```

### 场景：图片过大时前端拦截

```gherkin
  Scenario: 选择超过 20MB 的图片时前端拦截
    Given 我在采集页且会话有效
    When 我选择一张超过 20MB 的图片
    Then 我应该看到 "图片过大（最大 20MB）" 的提示
    And 不应该调用上传 API
```

### 场景：上传中显示进度且防重复提交

```gherkin
  Scenario: 上传过程中显示加载状态并阻止重复点击
    Given 我在采集页已选择一张图片并确认框选
    When 我点击 "确认上传"
    Then 该页面应该显示上传中的 loading 状态
    And 我再次点击 "确认上传" 时不应该产生第二个上传请求
```

### 场景：上传失败后可重试

```gherkin
  Scenario: 单页上传失败后只重试失败页
    Given 我已成功上传第 1 页
    And 第 2 页上传时网络异常导致失败
    When 我查看页面列表
    Then 第 2 页应该显示 "上传失败，请重试" 和重试按钮
    And 第 1 页不应该被重复上传
    When 我点击第 2 页的重试按钮
    Then 应该只重新上传第 2 页
```

### 场景：未采集任何页面时点击完成采集应提示

```gherkin
  Scenario: 零页时不允许完成采集
    Given 我在采集页且未上传任何页面
    When 我点击 "完成采集" 按钮
    Then 我应该看到 "请至少采集一页病历" 的提示
    And 不应该调用 finish API
```

### 场景：完成采集后页面切换为只读

```gherkin
  Scenario: 完成采集后锁定会话并提示到电脑端查看
    Given 我已成功上传 3 页病历图片
    When 我点击 "完成采集" 按钮
    Then 系统应该调用 POST /api/mobile/sess_001/finish
    And 页面应该切换为只读完成态
    And 我应该看到 "采集完成，请在电脑端继续审核" 的提示
```

### 场景：移动端视口适配

```gherkin
  Scenario: 移动端视口下按钮和交互区域不遮挡
    Given 我使用 iPhone 12 尺寸的移动设备 (390x844)
    When 我访问采集页
    Then 所有主要操作按钮应该在可视区域内
    And 按钮不应该相互遮挡
    And 四边形角点手柄应该可以单手拖动
```

## 5. Feature：页面列表与页序管理（PR-FE-002）

### 场景：多页上传后显示正确页序

```gherkin
Feature: 页面列表与页序管理
  作为 采集人
  我想要 管理已采集的页面顺序和内容
  以便 确保提交的病历页面顺序正确

  Scenario: 连续上传多页后页面按采集顺序编号
    Given 我在采集页依次上传了 3 张图片
    When 我查看页面列表
    Then 列表应该显示页序为 1、2、3
    And 每页应显示对应的缩略图
```

### 场景：删除页面后自动重新编号

```gherkin
  Scenario: 删除中间页面后剩余页面重新编号
    Given 页面列表中有 3 页 [A, B, C]
    When 我删除第 2 页
    And 我在确认弹窗中点击 "确认删除"
    Then 系统应该调用删除 API
    And 剩余页面应该重新编号为 1 (A)、2 (C)
```

### 场景：删除页面时取消操作不影响列表

```gherkin
  Scenario: 取消删除操作后列表保持不变
    Given 页面列表中有 3 页
    When 我点击第 2 页的删除按钮
    And 在确认弹窗中点击 "取消"
    Then 不应该调用删除 API
    And 页面列表应该保持 3 页不变
```

### 场景：拖拽调整页序

```gherkin
  Scenario: 拖拽调整页面顺序后调用排序 API
    Given 页面列表为 [A, B, C]
    When 我将 C 拖拽到最前面，顺序变为 [C, A, B]
    Then 系统应该调用排序 API 保存新顺序
    And 页面列表展示新顺序 [C, A, B]
```

### 场景：排序 API 失败时回滚

```gherkin
  Scenario: 排序接口失败时恢复原顺序并提示错误
    Given 页面列表为 [A, B, C]
    And 排序 API 将返回 500 错误
    When 我拖拽调整顺序
    Then 页面应该恢复到原来的顺序 [A, B, C]
    And 我应该看到 "排序失败，请重试" 的错误提示
```

### 场景：补拍页面插入指定位置

```gherkin
  Scenario: 在列表中间补拍新页面
    Given 页面列表中有 2 页 [A, B]
    When 我点击第 1 页和第 2 页之间的 "补拍" 入口
    And 拍摄一张新图片并上传
    Then 新页面应该插入到第 1 和第 2 之间
    And 列表重新编号为 1 (A)、2 (新)、3 (B)
```

### 场景：完成采集后页面列表只读

```gherkin
  Scenario: 锁定后所有编辑入口不可用
    Given 采集会话已锁定
    When 我查看页面列表
    Then 删除按钮应该不可见或禁用
    And 排序拖拽功能应该不可用
    And 补拍按钮应该不可见或禁用
```

## 6. Feature：四边形框选交互（PR-FE-009）

### 场景：图片预览后显示可调整的四边形

```gherkin
Feature: 四边形框选交互
  作为 采集人
  我想要 在拍照后调整四边形框选范围
  以便 排除屏幕背景、工具栏等非病历内容

  Scenario: 拍照后默认显示四边形 overlay
    Given 我已完成拍照并看到预览
    Then 图片上应该叠加一个四边形 overlay
    And 四边形的四个角点应该显示可拖动手柄
    And 默认角点应大约覆盖图片中心 80% 区域
```

### 场景：拖动角点更新坐标

```gherkin
  Scenario: 拖动左上角点后坐标和 overlay 同步更新
    Given 图片预览上显示了四边形 overlay
    When 我拖动左上角点向图片中心移动
    Then 状态中的 quad_points.tl 应该更新为新坐标
    And overlay 形状应该相应重绘
```

### 场景：角点不能拖出图片边界

```gherkin
  Scenario: 任一角点拖动不得超出图片边界
    Given 图片预览上显示了四边形 overlay
    When 我尝试将角点拖到图片边界之外
    Then 角点应该被限制在图片边界内
    And 坐标值不应包含负数或超出图片尺寸的值
```

### 场景：自相交四边形不可提交

```gherkin
  Scenario: 形成自相交四边形时阻止上传
    Given 我拖动角点导致四边形自相交
    When 我尝试点击 "确认上传"
    Then 上传应该被阻止
    And 我应该看到 "框选区域无效，请重新调整" 的提示
```

### 场景：未调整时使用默认范围上传

```gherkin
  Scenario: 不做任何调整直接确认上传使用默认范围
    Given 拍照后显示了默认四边形 overlay
    And 我没有拖动任何角点
    When 我直接点击 "确认上传"
    Then 上传请求体应该包含默认的 quad_points
    And 图片应该被成功上传
```

### 场景：用户手动调整后以上传手动坐标

```gherkin
  Scenario: 手动调整后提交的坐标以手动值为准
    Given 外部模块返回了自动识别的边界坐标
    And 前端展示了该自动边界
    When 我手动拖动角点调整范围
    And 我点击 "确认上传"
    Then 上传请求体中的 quad_points 应该是我手动调整后的坐标
    And 不应使用外部模块返回的自动坐标
```

### 场景：裁剪预览模块不可用时展示降级提示

```gherkin
  Scenario: 外部图像处理模块未配置时不伪造预览
    Given 外部图像处理模块未配置
    When 我调整四边形后点击 "预览裁剪结果"
    Then 我应该看到 "裁剪预览暂不可用" 的提示
    And 不应该展示伪造的矫正后图片
    And "确认上传" 按钮仍然可用
```

### 场景：overlay 渲染异常时有降级方案

```gherkin
  Scenario: 四边形 overlay 渲染出错时提供重新拍摄入口
    Given 图片预览页面加载时 overlay 组件渲染异常
    Then 错误边界应该接管并展示错误提示
    And 我应该看到 "重新拍摄" 的入口按钮
    And 不应该生成假的处理后的图片
```

## 7. Feature：任务列表（PR-FE-003）

### 场景：任务列表展示核心信息

```gherkin
Feature: 任务列表
  作为 医生
```

```gherkin

    When 我打开工作台

    # 5. 完成采集
    When 我点击 "完成采集"
    Then 会话锁定，提示到电脑端查看

    # 6. 任务处理
```

## 16. 实施建议

### 16.1 BDD 实施顺序（与 TDD 同步推进）

1. **工作台基础**：启动状态、空列表、二维码生成
2. **手机采集**：会话有效/无效/过期、拍照、选择图片、上传
3. **页序管理**：上传多页、删除、排序、补拍、锁定
4. **四边形框选**：overlay 展示、拖动、坐标校验、上传契约
5. **任务列表**：状态展示、新增、流转、筛选、失败态
6. **人工审核**：三区域布局、字段编辑、保存/回滚、确认、存疑
7. **来源核验**：来源展示、文本联动、无来源提示
8. **字段状态**：状态标签、统计、确认校验
9. **导出**：导出按钮、完整性检查、下载触发、失败恢复
10. **错误恢复**：连接失败、处理失败、保存失败、防重复
11. **离线安全**：无外部请求、断网可用、隐私保护

### 16.2 BDD 实施原则

- 每个 Scenario 实现前先确保对应的 TDD 用例已编写（RED）
- BDD 步骤定义应尽量复用，避免每个场景重复写 step 实现
- Gherkin 场景保持业务语言，不在 feature 文件中写实现细节
- E2E 场景使用 Playwright + MSW 组合执行
- 算法相关场景使用 fixture 数据，不依赖真实算法模块
